# AICUP2024‑Solar‑Energy‑Forecast

## 專案簡介  
本專案為「太陽能發電量預測」模型練習，旨在針對 AI 競賽 AI CUP 2024（依據區域微氣候資料預測太陽能發電量）所提供之資料，進行資料前處理、特徵工程、機器／深度學習模型訓練與預測流程。競賽資料為不同地點（17 個位置）之微氣候監測與光伏發電量記錄。  
你可以將本專案作為比賽參賽、研究範例或教學示範用途。

## 目錄結構  
```
AICUP2024‑Solar‑Energy‑Forecast/  
├── data/                    ← 原始資料／預處理資料資料夾  
├── notebooks/               ← Jupyter Notebook 分析與探索資料流程  
├── src/                     ← 程式碼（資料處理、模型訓練、預測腳本）  
├── models/                  ← 儲存訓練完成模型之所在位置  
├── submission/              ← 最終上傳檔案／結果檔案  
└── README.md                ← 本檔案  
```

## 執行環境與需求  
- 建議 Python 版本：3.8 或以上  
- 主要套件（可依實際程式補充）  
  - `numpy`  
  - `pandas`  
  - `scikit‑learn`  
  - `xgboost` or `lightgbm`（若使用）  
  - `tensorflow` / `pytorch`（若採用深度模型）  
  - `matplotlib`、`seaborn`（若有視覺化）  
- 建議使用虛擬環境（如 venv 或 conda）管理依賴  
- 若有 `requirements.txt`，請執行：  
  ```bash
  pip install ‑r requirements.txt
  ```

## 資料說明  
- 本次競賽所提供之訓練資料包含 17 個地點的微氣候變數（如：日照輻射、溫度、濕度、風速、降雨…）以及對應時間點的光伏發電量紀錄。  
- 預處理建議流程：  
  - 處理缺值、異常值  
  - 特徵轉換／衍生（如：日照累積、溫差、位置交互作用）  
  - 標籤（發電量）分割為訓練集／驗證集／測試集  
- 若專案中有目標時間範圍或地點指派，請於程式碼中說明。

## 模型訓練流程  
1. 載入並前處理資料  
2. 特徵工程（如 one‑hot 編碼地點、標準化／正規化）  
3. 模型選擇與訓練（如隨機森林、XGBoost、LightGBM、深度神經網路）  
4. 模型評估：使用指標如 MAE (平均絕對誤差)、RMSE (均方根誤差) 或比賽用之評分標準（例如絕對誤差總和）  
5. 儲存模型（如使用 `joblib`、`pickle`）以便後續預測使用  
6. 執行對測試集之最終預測，並產出可提交檔案（如 `.csv`）置於 `submission/` 資料夾。

## 如何執行  
假設程式名稱如下（請依實際檔案名稱調整）：

```bash
python src/predict.py             # 使用訓練完成模型進行預測與輸出提交檔案
```
